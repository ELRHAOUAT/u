De l'installation à l'exploitation sur Mutillidae II

Durée : 3h00 Niv : Technicien Spécialisé

Le site web vulnérable (Code Source).

Télécharger le ZIP

2. Installation et Configuration "Anti-Bug"

Attention : Suivez ces étapes scrupuleusement pour éviter les erreurs liées à PHP 8.
1
Installer XAMPP
Lancez l'installateur XAMPP. Installez-le à la racine par défaut : C:\xampp.
Une fois fini, ouvrez le XAMPP Control Panel.
2
Nettoyer la configuration PHP (Important !)
Dans XAMPP, cliquez sur le bouton Config (sur la ligne Apache) puis php.ini.
Une fenêtre Bloc-notes s'ouvre. Faites Ctrl+F pour trouver la ligne : 
display_errors = On
Remplacez On par Off pour obtenir : 
display_errors = Off
Enregistrez le fichier, fermez-le, puis cliquez sur Start devant Apache et MySQL.
3
Déployer Mutillidae
  1. Ouvrez le fichier ZIP mutillidae-master.zip téléchargé.
  2. Copiez le dossier contenu à l'intérieur vers C:\xampp\htdocs\.
  3. Renommez ce dossier en mutillidae.
  4. Allez dans C:\xampp\htdocs\mutillidae\ et supprimez le fichier .htaccess s'il existe.
4
Connecter la Base de Données
Ouvrez le fichier C:\xampp\htdocs\mutillidae\includes\database-config.php avec un éditeur de texte (Bloc-notes ou VS Code). Vérifiez les lignes suivantes : 
$dbHost = '127.0.0.1';
$dbUser = 'root';
$dbPassword = ''; // Laisser vide
$dbName = 'mutillidae4';

4. Attaque A : Login Bypass

Objectif : Se connecter en administrateur sans mot de passe.

Chemin : OWASP 2017 > A1 Injection > SQL Injection > Bypass Authentication > Login

La Procédure

  1. Allez sur la page de Login.
  2. Dans le champ Name, insérez le payload ci-dessous :
admin' OR 1=1 --
Attention : N'oubliez pas l'espace après les deux tirets !
Capture d'écran 2

Insérez ici la capture prouvant que vous êtes connecté ("You are logged in as admin").

5. Attaque B : Extraction de Données

Objectif : Voler la liste des utilisateurs et mots de passe via UNION SELECT.

Chemin : OWASP 2017 > A1 Injection > SQL Injection > Extract Data > User Info

Étape 1 : Identifier les colonnes

Injectez ceci dans le champ Name pour voir où les données s'affichent :

' UNION SELECT 1, 2, 3, 4, 5 --

Notez les chiffres qui s'affichent à l'écran (ex: 2 et 3).

Étape 2 : Le Dump Final

Injectez ce code pour extraire les données de la table accounts :

' UNION SELECT 1, username, password, 4, 5 FROM accounts --

6. Défense : Sécurisation du Code

En tant que développeur, vous devez corriger la faille dans le code PHP.

❌ Code Vulnérable

$user = $_POST['name'];
// Concaténation directe
$sql = "SELECT * FROM users WHERE n='" . $user . "'";
$res = $db->query($sql);

✅ Code Sécurisé (PDO)

$user = $_POST['name'];
// Requête préparée
$stmt = $pdo->prepare("SELECT * FROM users WHERE n=?");
$stmt->execute([$user]);
$res = $stmt->fetchAll();

Explication : Les requêtes préparées séparent la structure SQL des données, empêchant l'injection.
