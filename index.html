2. Exercice A : Login Bypass

Objectif : Se connecter sans connaître le mot de passe en manipulant la requête SQL.

La Procédure

1. Allez dans le menu : OWASP 2017 > Injection > SQL Injection > Bypass Auth > Login.

2. Dans le formulaire, injectez le code suivant :

admin' OR 1=1 --

(N'oubliez pas l'espace à la fin !)

Résultat Attendu

Si l'attaque réussit, voici ce que vous devriez voir à l'écran :

localhost/mutillidae/index.php?page=login.php

Login Page

You are logged in as Admin

3. Exercice B : Vol de Données

4. La Correction (Défense)

En tant que développeur, vous devez sécuriser le code PHP en utilisant PDO (Prepared Statements).

Code Vulnérable

Concaténation directe des variables.

$user = $_POST['name'];
$sql = "SELECT * FROM users WHERE n='" . $user . "'";
$res = $db->query($sql);

Code Sécurisé

Utilisation de `prepare` et `execute`.

$user = $_POST['name'];
$sql = "SELECT * FROM users WHERE n = ?";
$stmt = $pdo->prepare($sql);
$stmt->execute([$user]);
